# PyOrchestrator

## Overview

Full-stack Python Orchestrator web application (inspired by UiPath Orchestrator/Prefect). Manages Python automations across machines with queues, executions, schedules, projects, and real-time monitoring. Interface in Portuguese (Brazil).

pnpm workspace monorepo using TypeScript.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **Frontend**: React + Vite, Wouter routing, TanStack Query, shadcn/ui, Tailwind CSS, Recharts
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild

## Key Commands

- `pnpm --filter @workspace/api-server run dev` — start API server (port 8080)
- `pnpm --filter @workspace/orchestrator run dev` — start frontend (port 18842)
- `pnpm --filter @workspace/db run db:push` — push DB schema changes
- `pnpm --filter @workspace/db run db:seed` — seed database with sample data
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API client from OpenAPI spec

## Architecture

```
lib/
  api-spec/      - OpenAPI YAML spec
  api-zod/       - Zod schemas (generated by Orval)
  api-client-react/ - React Query hooks (generated by Orval)
  db/            - Drizzle schema + migrations

artifacts/
  api-server/    - Express 5 API server
  orchestrator/  - React + Vite frontend (previewPath: /)
  mockup-sandbox/ - Component preview server
```

## Pages (Frontend)

- `/` — Dashboard (metrics, charts, queue health, recent executions)
- `/machines` — Machine management (online/offline/maintenance status)
- `/projects` — Python project management (git/zip deploy)
- `/queues` — Execution queues with counts
- `/executions` — Execution history with status filtering
- `/executions/:id` — Execution detail with logs
- `/schedules` — Cron/interval/webhook schedulers
- `/users` — User management (admin/operator/viewer roles)

## Important Notes

- `lib/api-zod/src/index.ts` must ONLY contain `export * from "./generated/api"` — codegen overwrites and causes duplicate export errors otherwise
- All routes use `serialize()` helper before `.parse()` to convert Drizzle `Date` objects → ISO strings for Zod validation
- API server port: 8080, Frontend port: 18842
- Database seeded with 5 machines, 5 projects, 4 queues, 6 executions, 4 schedules, 5 users
